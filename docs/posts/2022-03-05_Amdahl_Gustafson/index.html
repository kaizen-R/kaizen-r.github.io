Gustafson‚Äôs Law

So once again, here is the Wikipedia entry for that.

How does that translate for me?

Well, consider parallelising in large and/or slow enough programs/scripts. Otherwise, it might not make sense.

To do so in R, I usually use these two approaches, in this particular order:

  • Use ‚Äúfutures‚Äù (and I personally focus usually on future_lapply()). This will create new processes that can run on other Cores of your CPU, while native R is single core single thread.

  • Use plumbeR, so that you can run your program over different CPUs/Servers/Containers altogether.
